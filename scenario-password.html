Scenario: Sicurezza delle Password

Le funzioni HASH servono a PROTEGGERE LE PASSWORD memorizzate nei database. Prendiamo in considerazione un possibile scenario:

  • CREAZIONE DELL'ACCOUNT: Si crea un account su un sito web. Durante la registrazione, si inserisce un nome utente e una password.

  • HASHING DELLA PASSWORD: La password è inviata al server che non la memorizzerà in chiaro nel database; Invece, userà una funzione HASH per TRASFORMARE la password in un HASH crittografico.

  • MEMORIZZAZIONE DELL'HASH NEL DATABASE: Il server memorizza solo l'HASH della password nel database. Quindi, se qualcuno ha accesso al database, non può vedere le password reali degli utenti.

  • ACCESSO ALL'ACCOUNT: Accedendo all'account e inserendo di nuovo la password, il server prende la password inserita, compie l'HASHING con la stessa funzione HASH e CONFRONTA L'HASH risultante con l'HASH memorizzato nel database.

  • CONFRONTO DEGLI HASH: Se l'HASH della password inserita corrisponde all'HASH memorizzato nel database, il server permette l'accesso all'account. Altrimenti, viene negato l'accesso.

Attacchi di Preimmagine

Un ATTACCO DI PREIMMAGINE è una tecnica utilizzata per trovare un input che corrisponda a un HASH dato. Questo attacco si divide in due categorie principali:

  • Preimmagine Semplice: Data una funzione HASH, si trova un INPUT che produce quello stesso HASH.
  • Seconda Preimmagine: Data un input e il suo HASH, si trova un INPUT differente che produce lo stesso HASH.

Le funzioni HASH SICURE sono progettate per rendere questi attacchi COMPUTAZIONALMENTE IMPOSSIBILI.

Come funziona un Attacco di Preimmagine?

La RESISTENZA ALLA PREIMMAGINE di una funzione HASH indica la difficoltà a trovare un INPUT per produrre uno specifico output HASH. La funzione è resistente se è difficile trovare i dati originali partendo dall'HASH

Gli attacchi di preimmagine cercano un input che produce uno specifico valore di HASH. L'attaccante ha un HASH di destinazione e trova un INPUT che produce quell'esatto HASH.

Es: Un utente ha una password HASHATA e l’attaccante cerca la password originale. L'attaccante, con l'attacco di preimmagine, cerca una password che produca lo stesso HASH.

Una funzione HASH sicura rende difficile trovare un input che produca uno specifico HASH.

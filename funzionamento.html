Qual'è il loro Funzionamento?

Le funzioni HASH trasformano, in maniera DETERMINISTICA, dati in un formato "HASH" UNIVOCO = IMPRONTA. Ecco come funziona un HASH:

  • L'algoritmo prende un INPUT (impronta del messaggio) e lo suddivide in BLOCCHI di dimensione fissa.
  • Ogni blocco è sottoposto ad OPERAZIONI MATEMATICHE, tra cui XOR, ROTAZIONI e SOSTITUZIONI.
  • L'output finale è un VALORE HASH DI LUNGHEZZA FISSA che rappresenta l'intero set di input.

Quali sono le Particolarità?

  • UNIDIREZIONALITÀ: Le funzioni HASH sono progettate per essere facili da calcolare in una direzione partendo dai dati originali. Tuttavia, il processo inverso è computazionalmente difficile.

  • COLLISIONI: Una collisione si verifica quando: due input diversi => stesso output HASH. Le buone funzioni HASH minimizzano le collisioni che compromettono la sicurezza.

  • UNIFORMITÀ DELLA DISTRIBUZIONE: Le funzioni HASH distribuiscono uniformemente l'output per evitare vulnerabilità e garantire sicurezza. L'assenza porterebbe alcuni HASH ad essere più probabili. Questo comporta aumento di COLLISIONI o aumento di ATTACCHI CRITTOGRAFICI.

  • UTILITÀ: Le funzioni HASH trovano sicurezza nei sistemi informatici.
